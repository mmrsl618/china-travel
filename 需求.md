# 管理后台重构需求

## 背景
生产流程改为：主编直接写英文 → 小二审核 → 视觉设计配图 → 小二审图 → 三哥检查 → 上线。
后台导航栏和文章状态需要适配新流程。

## 当前状态（旧的）
- `/admin/draft/zh` — 中文草稿
- `/admin/draft/en` — 英文草稿
- `/admin/done` — 已上线
- 状态流转：`zh_draft` → `zh_approved` → `en_draft` → `online`

## 新状态

### 导航栏
```
📄 待审草稿      → /admin/draft
🌐 待上站        → /admin/publish
📦 已上线        → /admin/done （不变）
```

### ① 待审草稿 /admin/draft
- 只显示 manifest 中 status=`zh_draft` 的文章
- 每行按钮：【编辑】+【通过】
- **编辑：** 读 `articles/{文章名}/source.html` 编辑，保存回 `source.html`（现有逻辑不动）
- **通过：** 
  - 读 `source.html` → 套母版模板（`apply_master_template`）生成 `article.html` 
  - manifest 状态改为 `en_draft`
  - **不执行 git push**（跟原来的 do_complete 不一样，原来的 complete 会 git push）
  - 成功后跳转到 `/admin/draft` 并显示提示

### ② 待上站 /admin/publish
- 只显示 manifest 中 status=`en_draft` 的文章
- 每行按钮：【预览】+【编辑】+【上站】
- **预览：** 启动本地浏览器打开 `http://localhost:8080/articles/{文章名}/article.html`（用 admin_server 自带的 preview 服务端口）
- **编辑：** 点编辑时先弹提示「此文章已套壳，如需修改请先撤回草稿，改完重新通过」，不提供编辑入口（C方案）
- **上站：**
  - 执行 `git add` + `git commit` + `git push`
  - push 成功后 manifest 状态改为 `online`，跳转到 `/admin/publish` 显示成功
  - push 失败时 manifest 不动（保持 `en_draft`），文章不消失，显示错误提示
- **搜索：** 待上站列表支持搜索（和草稿列表一样）

### ③ 已上线 /admin/done（不变）
- `/admin/done/{section}` 按板块查看已上线文章
- 功能按钮不变

### ✏️ 编辑功能
- 所有编辑功能**都是直接操作本地文件**（现有逻辑不变）
- 编辑窗口样式不变

### 状态流转

```
zh_draft  →  点通过（套壳，不push） →  en_draft  →  点上站（git push） →  online
                                                          ↑ push失败不动
             ↑ 撤回草稿，改完重新通过
```

### 删除废弃状态
- `zh_approved` 不再使用，manifest 中已有的可以改为 `zh_draft` 或 `en_draft`

## 注意事项
- 不要动 `admin_server.py` 以外的不相关代码
- editor 模板和前端 UI 样式不变
- 文档字符串和注释同步更新
- 确保 git push 失败时的状态处理正确（已修复过的bug不能复现）
