## 四、文章上线流程

### 流程图

```
小二推送稿件
     │
     ▼
┌──────────────┐
│  草稿管理     │  ← 所有未发布稿件列在这里
│  （.src/）    │
└──────┬───────┘
       │ 点击「✏️ 编辑」→ 用 contenteditable 编辑
       │ 改标题用 <h1>，正文正常写
       │ 底部「💾 保存内容」可随时保存
       │
       ▼
┌──────────────┐
│  确认完成      │  ← 点「✓ 完成」
│  自动处理：    │     1. 提取 <h1> 作为标题
│  ① 套壳模板    │     2. 剥离标题，正文不重复
│  ② git push   │     3. 套壳生成成品
│  ③ 标记上线    │     4. 自动 git 推送到 GitHub Pages
└──────┬───────┘
       ▼
┌──────────────┐
│  已上线       │  ← 文章出现在对应板块
│              │     可操作：
│              │     ✏️ 编辑 → 修改 → 💾 保存 → 🚀 重新发布
│              │     🗑 删除（同时删本地+线上）
└──────────────┘
```

### 详细操作步骤

#### ① 草稿 → 编辑
1. 打开 `http://localhost:8082/admin`，自动进入**草稿管理**页
2. 找到你要处理的稿件，点击 **「✏️ 编辑」**
3. 编辑页面说明：
   - 顶部有「所属板块」「子分类」「文件名」选项
   - 正文区可**直接编辑**（所见即所得），就像写 Word 文档
   - **用 `<h1>` 写标题**（选中文字，点格式选「标题1」），发布时自动提取
   - 正文里**不要再重复写标题**
   - 底部有两个按钮：**「💾 保存内容」**（随时保存）和 **「返回」**
4. 编辑完成后点击 **「💾 保存内容」**，会提示保存成功

#### ② 完成 → 发布上线
1. 回到草稿管理页，确认已保存
2. 点击该稿件的 **「✓ 完成」** 按钮
3. 系统自动做三件事：
   - **套壳**：提取 `<h1>` 作为页面标题，剥离正文中的标题，套入母版模板
   - **发布**：成品写入 `articles/xxx.html`
   - **同步**：git commit + push，自动部署到 GitHub Pages
4. 提示「已完成并发布上线」后，到对应板块就能看到文章

#### ③ 已上线文章维护
1. 点击左侧导航的板块（如 Before You Go），看到已发布文章列表
2. 每个文章有四个操作：
   - **✏️ 编辑** — 修改内容（编辑页面和草稿一样）
   - **👁 预览** — 在新标签页打开看效果
   - **🚀 重新发布** — 改完后重新套壳+同步
   - **🗑 删除** — 删本地文件 + 同步到线上移除
3. 如果板块有子分类，点击子分类（如 Visa Policies）可只看该分类的文章

## 五、编辑注意事项

### 标题规则
- ✅ **用 `<h1>` 写标题**，发布时自动提取到文章列表
- ❌ 正文内**不要重复写标题**，否则页面会出现两个标题

### 所属板块
- 编辑页顶部有「所属板块」下拉框，选错板块可以改
- 选板块后「子分类」下拉框会自动更新
- 子分类可选可不选

### 保存 vs 发布
- **「💾 保存内容」** = 存草稿，不上线（可多次保存）
- **「✓ 完成」** = 正式发布上线（草稿管理页的按钮）
- **「🚀 重新发布」** = 对已上线文章重新套壳+同步

### 删除操作
- 删除会同时：删本地 `articles/xxx.html` + 删 `.src/xxx.html` + 清 manifest + git push 到线上
- **删除不可恢复**，操作前会弹确认框

## 六、常见问题

**Q: 编辑页面没有编辑框？**
A: 检查浏览器控制台是否有报错。正常情况应看到一个可编辑的白色区域，带灰色边框。

**Q: 保存后页面刷新了但没看到文章上线？**
A: 保存只是存草稿，要回到草稿管理页点「✓ 完成」才会发布上线。

**Q: 重新发布后线上没更新？**
A: GitHub Pages 部署有1-2分钟延迟，稍等后刷新。也可以检查 git push 是否成功（后台会提示）。

**Q: 找不到某个文章？**
A: 如果文章已上线，在「✅ 已上线」对应板块找。如果没上线，在「📄 草稿管理」找。

**Q: 子分类选错了怎么办？**
A: 编辑页顶部的「子分类」下拉框随时可以改，保存后生效。

## 七、技术说明（给程序员）

### 关键文件

| 文件 | 说明 |
|------|------|
| `admin_server.py` | 管理后台 v3（~733行），Flask 风格路由 |
| `templates/article-master.html` | 套壳母版，含 `__TITLE__` `__DESCRIPTION__` `__CONTENT__` `__NAV__` 四个变量 |
| `articles/.manifest.json` | JSON 清单，每条含 `title` `section` `sub_category` `status` |

### 核心函数

| 函数 | 作用 |
|------|------|
| `apply_master_template()` | 套壳：替换母版中的变量，生成完整 HTML |
| `extract_title()` | 从 `<h1>` 提取标题 |
| `strip_title()` | 剥离 `<h1>`，正文不重复显示 |
| `git_push()` | git add → commit → push |
| `sidebar_nav()` | 生成左侧导航 HTML（含二级折叠 JS） |

### 路由

| 路由 | 方法 | 功能 |
|------|------|------|
| `/admin` | GET | 重定向到草稿管理 |
| `/admin/draft` | GET | 草稿列表 |
| `/admin/done/{section}` | GET | 板块文章列表（可选 `?sub=k` 过滤子分类） |
| `/admin/edit?path=&from=` | GET | 编辑页 |
| `/admin/save` | POST | 保存源文件 |
| `/admin/complete` | POST | 完成→套壳→上线 |
| `/admin/republish` | POST | 重新发布（重套壳+同步） |
| `/admin/delete` | POST | 删除本地+线上 |

### 启动方式

```bash
cd E:\项目库\china-travel-website
python admin_server.py
# 后台运行在 http://localhost:8082/admin
```
