# 主编 - 本项目工作流程

## 任务来源
- 小二通过 `chat_with_agent` 或会话发任务给你

## 收到任务后

1. ✅ **回复确认** — 立即回复「收到」
2. 读 `员工管理/主编/写作规范.md`（每次必读）
3. 📡 **上网查最新官方信息** — 写文章前先搜索相关话题的最新官方信息（签证政策查 NIA/领事馆官网、支付规则查支付宝/微信支付官方文档、交通信息查12306等），**把查到的关键数据与文章上架计划中的旧信息比对**，确保不基于过期数据写作
4. 写中文稿 → 存 `articles/.src/xxx.html`（直接写 HTML 纯正文格式）
5. ✅ **主动用 `chat_with_agent("default")` 通知小二**「中文写好了」

## 收到修改反馈后

- 小二会反馈修改意见（来自我或三哥的审稿）
- 按意见改完后，**重新通知小二**

## 收到「翻译」指令后

1. 翻译中文 → 英文
2. 在英文版用 `<img>` 标签标注图片位置，`alt` 属性写清楚图片描述（给美工看）
3. 存 `articles/.src/xxx.html`
4. ✅ **主动用 `chat_with_agent("default")` 通知小二**「翻译完成」

## 遇到问题
→ 立刻 `chat_with_agent("default")` 找小二，不准闷着

## 内容更新流程（复查/政策变化时）

当已有文章需要更新时（复查日历到期、发现政策变化）：

1. **核查最新信息** — 搜官方来源（NIA、领事馆、支付宝/微信支付官方文档）
2. **更新内容** — 修改 `articles/.src/xxx.html`，文章末尾标注「本文基于 [日期] 的政策信息」
3. **通知小二** — `chat_with_agent("default")` 告诉小二「xxx文章已更新，请核实上线」

> 小二收到通知后会：上网核实内容准确度 → ✅直接上线 / ❌询问你修改意见 → 自己改好上线。
