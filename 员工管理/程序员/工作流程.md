## 管理后台核心逻辑（admin_server.py）

### 四界面工作流
```
📄 中文待审 → 📝 英文待审 → 📦 已发布(本地) → ✅ 已上线(GitHub Pages)
```

### 关键函数

| 函数 | 功能 |
|------|------|
| `extract_article_content(full_html)` | 从英文审稿HTML提取 title/description/正文 |
| `apply_master_template(content, title, desc, section)` | 套母版（替换 `__TITLE__` 等变量），图片路径修正 `../../images/` → `../images/` |
| `sync_guide_link(source_path, article_file, title)` | ✓完成时自动在 guides 页对应 tab 插入链接 |
| `get_published_set()` | 读取 `.publish-state.json` 获取已上线文章列表 |
| `mark_published(fname)` | 标记文章为已上线（写入 `.publish-state.json`） |
| `_do_publish(article_rel, article_name, section_key)` | git push 单篇文章到 GitHub Pages |
| `_do_delete(fname, section_key)` | 删除文章（git rm + push + 去链接 + 取消上线标记） |
| `_do_sync()` | 全量同步（自动备份标签 → add → commit → push） |

### 图片路径处理
- 源文件路径：`articles/en/before-you-go/visa/xxx.html` → 用 `../../../../images/xxx`
- 成品路径：`articles/xxx.html` → 用 `../images/xxx`
- 套壳时自动替换：`../../images/` → `../images/`（在 `apply_master_template` 中处理）

### 已发布/已上线状态管理
- 文章列表从 `guides/*.html` 解析 `<a class="article-link">`
- 状态存在 `articles/.publish-state.json`（未跟踪文件）
- 已发布 = 文件在 guides 中但不在 published set 中
- 已上线 = 文件在 guides 中且在 published set 中

### 编辑页说明
- 编辑页使用 `contenteditable="true"` 富文本编辑
- 如果显示 HTML 代码（标签被转义），检查 `render_edit_page` 中的 `.replace('<', '&lt;')` 是否被错误地启用了
- 正确做法：直接放 HTML 内容到 `contenteditable` 里

## Git 配置

```python
GIT_EXE = r'C:\Program Files\Git\bin\git.exe'  # 不在PATH中
# 代理（国内访问GitHub）
git config http.proxy socks5://127.0.0.1:10808
```

### 常用操作
- **上线单篇文章**：`_do_publish()` → add + commit + pull --rebase + push
- **全量同步**：`_do_sync()` → tag + push tag + add all + commit + pull --rebase + push
- **删除文章**：`_do_delete()` → rm + commit + pull --rebase + push + 去链接 + 取消标记
- **文件未跟踪处理**：删除前先 `git ls-files --error-unmatch` 判断，跟踪走 `git rm`，未跟踪走 `os.remove()`
