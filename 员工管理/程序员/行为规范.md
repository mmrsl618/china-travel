# 💻 程序员 - 行为规范

> 管理后台、Git 运维、前端代码维护相关规范。
> 所有代码相关操作前，**必须先读此文件**。

## 核心原则

1. **先读再改** — 任何代码改动前，先读 `PROJECT.md`（项目文件）+ `admin_server.py`（管理后台源码）+ 对应规范文件
2. **不私自上线** — 代码修改后需要三哥确认才能部署
3. **破坏性操作先问** — 删除文件、修改结构、改配置前先问小二或三哥
4. **代码纪律** — 不改模型配置、不用 PowerShell 处理文本（编码会损坏）

## 技术栈红线

| 约束 | 说明 |
|------|------|
| **纯静态** | 无服务器、无数据库、无后端框架 |
| **无框架** | 纯 HTML/CSS/JS，不用 React/Vue 等 |
| **GitHub Pages** | 仅限静态站点，不支持服务端渲染 |
| **一套 HTML 自适应** | 不是两套页面，CSS 响应式处理 |
| **Python 管理后台** | 仅本地使用，不上线 |
| **电脑只有 Edge** | 没有 Chrome，浏览器调试用 Edge |

## 代码质量标准

- 修改 `admin_server.py` 后必须 `py_compile.compile()` 检查语法
- 修改 `style.css` 后必须在 8080 预览检查 PC + 手机（768px断点）效果
- 修改 `js/main.js` 后必须在 Edge 中测试功能
- 所有文件使用 UTF-8 编码

## Git 纪律

- GIT_EXE = `C:\Program Files\Git\bin\git.exe`（不在 PATH 中）
- Git 代理：`http.proxy=socks5://127.0.0.1:10808`
- 工作流：`add → commit → pull --rebase → push`
- **首推新项目必须用 `push --force`，不用 `pull --rebase`**（rebase 冲突可能覆盖本地文件）
- 同步前自动创建时间戳备份标签（`auto-YYYYMMDD_HHMM`）
- 每次同步前必须先备份（自动标签）
