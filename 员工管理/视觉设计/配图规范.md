# 配图规范 — china-travel-website

> 视觉设计每次开工前第一件事必须读取此文件。

## 配图来源

### 手机App/界面截图类（竖版9:16）— 不再使用
⚠️ **新规：所有配图统一为16:9横版场景图，不再使用手机界面/APP截图类配图。** 搜图规则保留为备用。

### 旅游景点实拍图（横版16:9）
**用浏览器搜图截图**。流程：
- 搜图 → 找到对应景点的高清实拍照片
- 有水印/LOGO的 → **去水印后再截取**
- 裁剪为16:9横版
- 文件大小保持 ≤200KB

### 场景/概念类图（横版16:9）
**用生图技能**，调用 `Agnes Image 2.0` 模型。生图提示词按以下结构编写：

**生图提示词结构：**
```
[风格] + [景别] + [构图方式] + [视角] + [光影色调] + [品质]
```

**固定要求：**
- 风格：写实风格
- 品质：8k超清品质

**示例：**
```
写实风格，全景，居中构图，平视，自然光暖色调，8k超清品质，a traveler standing on a busy Shanghai street...
```

内容部分从主编的 `<img alt="...">` 描述中提取，拼接到结构后面。

## 基本原则

1. **严格按主编 `<img alt="...">` 的描述配图**
2. **图片宽度与比例：**
   - 🖼️ **统一16:9横版** → 内联style写 `max-width:100%;display:block;margin:20px 0`，宽度自适应
   - 高度按比例自动计算
3. **文件大小：** 单张 ≤ **200KB**
4. **写实风格 + 8k超清质感**，禁止插画/卡通/3D渲染

## 自检清单（交图前必查）

每张图生成后、嵌入文章前，逐项检查。发现问题自己改，全部通过才嵌入。

| # | 检查项 | 说明 |
|---|--------|------|
| 1 | ❌ 水印/LOGO/角标 | 平台logo、截图角标、版权声明等，一票否决。app截图如有水印→去水印后再截取 |
| 2 | ❌ 人物/动物结构异常 | 多手指/多手脚/残缺/肢体错位，一票否决 |
| 3 | ✅ 图片类型正确 | 统一16:9横版场景/实拍图，无AI乱码文字 |
| 4 | ✅ 内容匹配主编alt描述 | 图片内容与文章中的 `<img alt="...">` 一致 |
| 5 | ❌ 文字无乱码/重叠/超框/截断 | 每个字符完整可读，不互相覆盖 |
| 6 | ❌ 场景穿帮 | 人物/物件与背景协调，不悬空 |
| 7 | ✅ 尺寸比例正确 | 统一16:9横版 |
| 8 | ✅ 图片宽度+居中 | `max-width:100%;display:block;margin:20px 0` |
| 9 | ✅ 文件大小 ≤200KB | |
