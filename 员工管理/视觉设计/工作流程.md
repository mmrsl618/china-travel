# 视觉设计 - 本项目工作流程

## 任务来源
- 小二通过 `submit_to_agent` 或 `chat_with_agent` 发任务给你

## 收到任务后

1. ✅ **回复确认** — 立即回复「收到」
2. 读 `员工管理/视觉设计/配图规范.md`（每次必读）
3. 📖 **读任务指定的文章** — 从文章中找到主编标注的 `<img alt="...">` 描述，确定需要配什么图

## 生图流程（全新文章配图）

### 场景/摄影类（横版16:9）

```
① 生图 → 用 skills/生图/ 技能包生成图片
② 嵌入文章 → 把图片放入文章中主编标注的 <img> 标签位置
③ 自查自检 → 按配图规范中的「自检清单」逐项检查
④ 自改 → 发现问题的，自己修改后重新回到②，直到全部通过
```

### 手机App截图类（竖版9:16）

```
① 搜图 → 用浏览器搜与实际app一致的真实截图
② 去水印 → 有平台logo/水印的，去水印后再截取
③ 截图保存 → 存到文章 images/ 目录，文件名保持规范
④ 嵌入文章 → 把图片放入文章中主编标注的 <img> 标签位置
⑤ 自查自检 → 按配图规范中的「自检清单」逐项检查
⑥ 自改 → 发现问题的重新搜图，直到全部通过
```

**⚠️ 效率要求：** 生图任务超时上限20分钟。
- 自检不需额外压缩时间，文件大小限制由生图API自动控制
- 如果20分钟还不够，立刻 chat_with_agent("default") 报告，不准闷着拖到超时

## 修改已有图片流程（用于重做/修复/替换现有配图）

此流程适用于小二或三哥要求修改文章中已存在的某张图。

```
① 确认要改哪张图（文件路径 + 文件名）和改什么
② 生图 → 用 skills/生图/ 技能包生成新图，覆盖保存到原路径（文件名不变）
③ 同步更新 source.html → 修改该 <img> 标签的 alt 属性，与最新图片内容一致
④ 自查自检 → 按配图规范中的「自检清单」逐项检查
⑤ 自改 → 自检不过的，重新生图回到②
```

**⚠️ 关键：修改图片时，文件名必须保持不变**（否则文章中的引用会失效）。
**⚠️ 关键：改图后必须同步更新 source.html 中的 alt 描述**（否则描述与图片不匹配）。

## 配图完成后

- ✅ **主动用 `chat_with_agent("default")` 通知小二** — 不管任务完成/失败/超时/被取消都要通知
- ❌ 不准用 `qwenpaw channels send`（小二收不到）

## 收到小二的修改意见后

- 小二会告诉你哪张图、什么问题、怎么改
- 按意见修改后，**重新执行对应的流程**（新配图走生图流程，修改走修改已有图片流程）
- 改完后再次 chat_with_agent("default") 通知小二

## 遇到问题
→ 立刻 `chat_with_agent("default")` 找小二，不准闷着
