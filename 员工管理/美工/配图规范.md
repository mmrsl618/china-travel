# 图片规范 — china-travel-website

> **美工每次开工前第一件事必须读取此文件。**

## 配图来源（唯一方式）

**只准用 `skills/生图/` 技能包生图**（位于美工工作区 `skills/生图/`），调用 Agnes Image `agnes-image-2.1-flash` 生成写实摄影风格图片。

### 调用方式（通过 execute_shell_command）

```bash
# 手机截图/APP界面（9:16 竖屏）
python skills/生图/scripts/generate.py screenshot --prompt "英文描述" --output "{项目根路径}/images/文件名.jpg"

# 场景摄影/街景/建筑（16:9 横屏）
python skills/生图/scripts/generate.py scene --prompt "英文描述" --output "{项目根路径}/images/文件名.jpg"

# 图标/头像/方形图（1:1）
python skills/生图/scripts/generate.py square --prompt "英文描述" --output "{项目根路径}/images/文件名.jpg"
```

⚠️ **`--output` 必须用绝对路径**，格式：`{项目根路径}/images/文件名.jpg`
⚠️ **严禁使用 write_file 写图片文件。** 生图脚本自动存图 + 自动压缩 ≤200KB
⚠️ **严禁使用浏览器搜图、截图、或其他任何方式获取图片。** 发现一次直接上报三哥。

**配图时间控制：** 单篇文章总配图时间控制在 **5分钟内**（含生图+可能的搜图），不准超时。

## 基本原则

1. **严格按主编的描述配图**
2. **图片比例和尺寸：**
   - 📱 **手机界面截图** → 竖屏 **9:16**，`max-width: 400px`
   - 🖼️ **其他图片（场景/摄影等）** → 横屏 **16:9**，`max-width: 600px`
   - 所有图片 **居中**（`display: block; margin: 20px auto`）
3. **文件大小：** 单张 ≤ **200KB**，保证页面加载速度
3. **文字必须完整可读** — 无重叠、无遮挡、无截断
3. **文字对比度足够** — 不因颜色过浅而难以辨认
4. **布局整洁** — 同类元素对齐，互不干扰
5. **不加多余标题/标注**

## 审图标准

```
① 内容匹配主编描述
② 类型匹配（路线图/流程图/示意图）
③ 所有文字完整显示
④ 无文字重叠
⑤ 文字对比度足够
⑥ 布局整洁，互不干扰
⑦ 无多余/截断的标注
⑧ UI 穿帮检查 — 键盘/按钮等 UI 元素与背景/设备框体对齐，无脱离
⑨ 手机截图类 — 文字/内容不超出屏幕边界
```

## 生图质量专项检查（AI 生成图特别留意）

- ✅ **写实风格** — 所有生成图片必须是真实摄影写实风格，禁止插画/卡通/3D渲染
- ❌ **水印** — 严格检查：图片上不得有任何水印、LOGO、文字标注、平台角标
- ❌ **文字重叠** — 不同字段的文字不能互相覆盖
- ❌ **UI 穿帮** — 键盘按键与屏幕框体/设备实体对齐，不能悬空或错位
- ❌ **超框** — 手机屏幕截图/APP 界面类图片，文字不能超出屏幕边界
- ✅ **文字完整** — 每个中/英文字符完整显示，不截断、不遮挡
