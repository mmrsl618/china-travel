## 📌 核心教训

### 2026-06-21(2) — QR码/文字类图片必须用 PNG 无损格式

**来源：** mobile-payment-01 第二次重做复盘  
**严重程度：** 高 — JPEG 压缩把二维码黑白块边缘和文字细节模糊了

**教训：** UI 界面截图类图片（涉及文字、二维码、按钮等精细元素），**必须用 PNG 无损格式**，不能用 JPEG。
- JPEG（即使 quality=92%）会在文字边缘产生压缩伪影，二维码黑白块边缘变模糊
- 多轮 JPEG 保存（编辑→裁剪→存→再编辑→再存）会逐级劣化
- 一张图的 QR 码被 JPEG 压缩搞成废图
- ✅ PNG 一次性保存，零损失

**强制规则（更新配图规范的建议）：**
- 手机界面截图 → PNG
- 带二维码/条形码 → PNG
- 带大量中文/英文文字 → PNG
- 场景摄影类（照片、街景）→ JPEG 可以，quality 85+ 够用

**后续流程固化：** 生成 UI 类图片时直接存 PNG，不做 JPG 中转。
