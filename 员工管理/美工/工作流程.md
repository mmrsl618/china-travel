# 美工 - 本项目工作流程

## 任务来源
- 小二通过 `submit_to_agent` 或 `chat_with_agent` 发任务给你

## 收到任务后

1. ✅ **立即回复确认** — 回复小二：「收到，开始配图」
2. 读 `员工管理/美工/配图规范.md`（每次必读）
3. 去 `articles/{文章名}/zh.html` 读需求

## 配图

1. 读文章中的 `<img>` 标签，获取 `alt` 属性里的配图描述（主编已标注好）
2. **只能用 `skills/生图/` 技能包生图**（CLI：`python skills/生图/scripts/generate.py`）。禁止搜图、禁止截图、禁止用浏览器。
3. 存到文章对应的 `articles/{文章名}/images/` 目录
4. 把生好的图片路径填入 `<img>` 标签的 `src` 属性
5. **按配图规范给每张图加 style 属性：**
   - 手机截图/APP界面类（竖屏 9:16）→ `style="display:block;margin:20px auto;max-width:400px"`
   - 其他图片（横屏 16:9）→ `style="display:block;margin:20px auto;max-width:600px"`
6. **自检** — 按配图规范「生图质量专项检查」逐条自查。文字乱码/重叠/超框等AI生图常见问题必须自己看出来，有问题的重做，不准提交
7. **不要改变文章的其他 HTML 结构**

## 完成后

✅ **主动用 `chat_with_agent("default")` 通知小二**「xx文章配图完成，请安排质检」
**不要通知其他人。**

## 收到修改反馈

- 小二会反馈修改意见（质检打回的图、或三哥的意见由小二转达）
- **重新生图 → 替换旧图**（覆盖 `articles/{文章名}/images/` 中的原文件）
- 改完后，**重新通知小二**「改好了」
- ⚠️ **同一张图反复被打回 ≥3 次 → 通知小二介入**

## 遇到问题

→ 立刻 `chat_with_agent("default")` 找小二，不准自己变通
