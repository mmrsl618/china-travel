## 一、竞品分析

### 1. 头部旅行商（China Highlights / Travel China Guide）

| 特点 | 说明 |
|------|------|
| 内容量 | 数百到数千篇文章，覆盖每个城市、每个话题 |
| 模式 | 内容 → 询单 → 卖旅行团 |
| 起步 | 运营 20+ 年 |
| 优势 | SEO 权重极高，Google 前排全是他们 |
| 劣势 | 内容偏推销、界面老旧、信息过载 |

**我们的机会：** 他们不是对手，他们是旧时代的恐龙。我们不卖团、不堆 SEO 垃圾，做"一个干净、有用、读完就能自己走的网站"——这在中文旅游内容领域几乎不存在。

### 2. 独立内容创作者（The Dana Edition / Sam Blinson / Maverickbird）

| 特点 | 说明 |
|------|------|
| 结构 | 一篇"终极指南"母文章 + 城市/专题深度文章 |
| 风格 | 第一人称、实操经验、痛点驱动 |
| 流量 | SEO + Pinterest + Reddit |
| 变现 | 联盟营销（Airalo VPN、eSIM、Trip.com、Amazon 装备） |
| 内容量 | 10~30 篇文章即可建立信任 |

**关键发现：** The Dana Edition 只有约 15 篇中国相关文章，却做到了 Google 首页。凭的就是"一篇文章把一个事讲透"。

### 3. 我们的定位

| 维度 | 他们 | 我们 |
|------|------|------|
| 目标 | 卖团 / 赚佣金 | 先建信任，后变现 |
| 内容 | 推销感重 / 偏个人 | 实用工具型，去个人化 |

## 二、内容分类体系

### Type A · 搭建型（一次上架，长期稳定）
适用话题：签证政策、支付基础、VPN 设置、入境流程。这些内容几乎不变，写一次管三年。偶尔微调即可。

### Type B · 成长型（持续更新，无限扩展）
适用话题：城市指南、行程规划、美食推荐、交通实操。新城市、新路线、新季节内容可无限延伸。

## 附：新版文件夹结构（2026-06-23 起）

每篇文章独立文件夹，不再统一放 `articles/.src/`：

```
articles/
├── .manifest.json               ← 文章元数据（标题/板块/状态）
├── .publish-state.json          ← 已上线文章列表
├── china-tourist-visa-l-visa/   ← 示例：L-Visa 文章
│   ├── zh.html                  ← 源文件（英文翻译版，含 <img> 配图标签）
│   ├── en.html                  ← 套壳后的上线版本（套 article-master.html 模板）
│   └── images/                  ← 文章专属配图
│       ├── 01-l-visa-in-passport.jpg
│       ├── 02-visa-free-countries-map.jpg
│       ├── 03-cova-system-screenshot.jpg
│       └── 04-application-documents.jpg
├── mobile-payment-in-china/
│   ├── zh.html                  ← 源文件（英文版含 img 标签）
│   ├── en.html                  ← 已上线版本（含完整 HTML 外壳）
│   └── images/
│       └── ...

```

**文件角色说明：**
- `zh.html`：文章内容文件（写作→中文审稿→翻译后替换为英文版+配图标签）
- `en.html`：套 `templates/article-master.html` 模板后的完整 HTML，可直接上线
- `images/`：该文章专属配图，由美工产出

## 八、新板块建议

### 建议 1：拆分 "Food & Dining"

当前没有独立的饮食板块。饮食是旅游第二大决策因素（仅次于目的地选择）。建议：

- 新板块：**"Food & Dining"**
- 位置：放在 Explore China 旁边或作为其子板块
- 首期文章：
  - How to Order Food in China（扫码点餐、菜单翻译）
  - China Street Food Guide
  - 按城市的美食指南（北京烤鸭、成都火锅、西安面食）

### 建议 2：新增 "Culture & Etiquette"

很多外国人因为不知道中国礼仪而焦虑。这个板块可以解决"社交安全感"问题：

- 新板块：**"Culture & Etiquette"**
- 首期文章：
  - Chinese Etiquette: Dos and Don'ts
  - How to Use Chinese Toilets（这是真实痛点，Google 搜索量很高）
  - Tipping in China

### 建议 3：新增 "News & Updates"

政策变化（签证免签扩容、支付规则调整）需要一个快速发布渠道：

- 新板块：**"Updates"**（放在首页或独立页）
- 形式：短文章（800~1500 字），快速发布
- 作用：SEO 时间敏感性内容 + 展示网站活跃度

> 这三个建议**非 MVP 必须**，但如果 MVP 效果好，Phase 2 优先启动 Food & Dining。

## 十二、风险与自检

### 风险 1：空 Tab 导致跳出率高
→ **对策**：MVP 必须 16 篇全上，宁可推迟上线也不空 Tab 上线。

### 风险 2：内容不够深，被竞品碾压
→ **对策**：每篇 3000~4000 字中文，对标竞品顶级文章。不做"10 个 tips"类浅文。

### 风险 3：Google 不收录新站（沙盒期）
→ **对策**：上线后立刻提交 sitemap，Reddit/Pinterest 引流加速收录。沙盒期通常 3~6 个月，期间靠社交媒体扛。

### 风险 4：签证/政策文章快速过时
→ **对策**：每篇末尾加"Last updated: 月份 年份"，建立每季度复查机制。检测到政策变化 48 小时内更新。

### 风险 5：城市指南无差异化
→ **对策**：每条指南必须有"只有住过才知道"的细节，不是维基百科翻译。用地铁路线、区域选择、真实价格来建立信任。

### 风险 6：变现太急破坏信任
→ **对策**：MVP 阶段零变现。Phase 2 只嵌入"自然出现"的推荐（游客本来就需要买 eSIM/VPN），不做硬广。

### 自检问题

1. **访客打开首页，3 秒内知道这个网站是干什么的吗？** → 需要 Hero Section 明确定位
2. **访客能一键找到签证信息吗？** → Before You Go 必须是最显眼的入口
3. **每篇文章看完后，访客知道下一步看什么吗？** → 结尾必须有引导链接
4. **如果我是第一次去中国的老外，这个网站能解决我 80% 的焦虑吗？** → 这是 MVP 的最高标准
5. **网站有"惊喜感"吗？** → 别人没写过但人人需要的细节（比如厕所指南、点餐话术、中文地址保存方法）
